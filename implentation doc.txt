### Phase 3: The FastAPI Backend
**Goal:** Serve as the recommendation engine. It takes the array of scanned ingredients, merges them with the user's "Virtual Pantry," and calculates overlap scores.

**1. Project Structure**
```text
backend/
├── main.py
├── recipe_db.json
├── models.py
└── requirements.txt
```

**2. Core Implementation (`main.py`)**
Keep this stateless. Load the JSON file into memory on startup.

```python
from fastapi import FastAPI
from pydantic import BaseModel
import json

app = FastAPI(title="Recipe Recommender API")

# Load DB into memory on startup
with open("recipe_db.json", "r") as f:
    RECIPE_DB = json.load(f)

class RecipeRequest(BaseModel):
    scanned_ingredients: list[str]
    pantry_ingredients: list[str]
    diet_preference: str | None = None

def calculate_jaccard(user_set, recipe_set):
    intersection = len(user_set.intersection(recipe_set))
    return intersection / len(recipe_set) if len(recipe_set) > 0 else 0

@app.post("/api/recommend")
async def get_recommendations(req: RecipeRequest):
    user_ingredients = set(req.scanned_ingredients + req.pantry_ingredients)
    recommendations = []

    for recipe in RECIPE_DB:
        # Apply hard filters first (e.g., Vegetarian)
        if req.diet_preference and recipe['diet'] != req.diet_preference:
            continue
            
        recipe_ing_set = set(recipe['ingredients'])
        score = calculate_jaccard(user_ingredients, recipe_ing_set)
        
        # Threshold: User has at least 70% of the required ingredients
        if score > 0.70:
            missing = list(recipe_ing_set - user_ingredients)
            recommendations.append({
                "recipe": recipe,
                "score": score,
                "missing_ingredients": missing
            })

    # Sort by highest match score
    recommendations.sort(key=lambda x: x['score'], reverse=True)
    return {"results": recommendations[:10]} # Return top 10
```
* **Deployment:** For early testing, running this locally (via WSL or your standard Linux terminal) using `uvicorn main:app --reload --host 0.0.0.0` will allow your physical phone to hit the local network IP.
